Accessing Resources

Using Access Tokens

Once the client application has an access token, requests can be issued passing the access token in the request headers, or as a request parameter.

To pass the access token via request header, set the authenticate header as such:

Authorization: Bearer ACCESS_TOKEN

Alternatively, you can add it as a request parameter:

http://COMMUNITY/path/to/protected/resource?bearer_token=ACCESS_TOKEN

Successful Response

If authorization is successful, the server will return an HTTP 2xx or 3xx response code and a response body set to the requested resource.

Error Response

If authorization fails, the server will return an HTTP 401 response code.  If you are nearing your access token's expiration time, you may be able to avoid future 401 errors by obtaining a new access token. See "Maintaining Access: Using Refresh Tokens" for more information.
